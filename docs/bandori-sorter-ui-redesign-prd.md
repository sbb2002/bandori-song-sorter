# PRD: bandori-album-sorter UI 개편

## Problem Statement

현재 bandori-album-sorter는 **앨범 단위**로 소팅을 수행한다. 이는 팬들이 실제로 선호도를 판단하는 단위인 **개별 곡**과 괴리가 있다. 앨범에는 여러 곡이 포함되어 있으므로, 앨범 전체를 하나의 단위로 평가하면 곡별 미묘한 선호 차이를 반영하기 어렵다.

또한 소팅 결과를 타인과 공유할 때 활용 가능한 형식이 제한적이며, DC인사이드 등 커뮤니티 공유 채널과의 연동이 부족하다.

## Solution

소팅 단위를 앨범에서 **곡(Song)** 으로 전환하고, 밴드별 곡 리스트 → 랭크 팝업의 인터랙션 플로우를 도입한다. 랭크 시각화는 밴드별 히스토그램과 전체 히트맵으로 이원화하며, DC인사이드 커뮤니티에 최적화된 공유 기능을 추가한다.

## User Stories

### 소팅 인터랙션

1. As a 밴드리 팬, I want to 밴드 아이콘을 클릭하면 해당 밴드의 곡 리스트가 펼쳐지길 바란다, so that 앨범이 아닌 개별 곡 단위로 소팅을 시작할 수 있다.
2. As a 밴드리 팬, I want to 곡 항목을 클릭하면 랭크 팝업이 뜨길 바란다, so that 직관적으로 해당 곡의 선호도를 입력할 수 있다.
3. As a 밴드리 팬, I want to 랭크 팝업에서 최애 / 차애 / 호 / 중간 / 불호 중 하나를 선택할 수 있길 바란다, so that 기존 세분화된 티어보다 직관적인 5단계 감성 분류를 쓸 수 있다.
4. As a 밴드리 팬, I want to 이미 랭크를 매긴 곡은 시각적으로 구분되길 바란다, so that 어떤 곡을 아직 평가하지 않았는지 한눈에 확인할 수 있다.
5. As a 밴드리 팬, I want to 소팅 도중 언제든지 기존 랭크를 수정할 수 있길 바란다, so that 의견이 바뀌어도 처음부터 다시 하지 않아도 된다.
6. As a 밴드리 팬, I want to 모바일에서도 동일한 팝업 방식으로 랭크를 입력할 수 있길 바란다, so that 기기에 관계없이 일관된 경험을 가질 수 있다.

### 시각화

7. As a 밴드리 팬, I want to 밴드별 히스토그램에서 해당 밴드 곡들의 티어 분포를 볼 수 있길 바란다, so that 내가 어떤 밴드를 전반적으로 좋아하는지 파악할 수 있다.
8. As a 밴드리 팬, I want to 전체 히트맵을 통해 모든 밴드에 걸친 소팅 결과 요약을 볼 수 있길 바란다, so that 밴드 간 선호도를 비교할 수 있다.
9. As a 밴드리 팬, I want to 기존 앨범 자켓에 표시되던 랭크 칸이 제거되길 바란다 (PC/모바일 모두), so that 화면이 더 깔끔해지고 곡 단위 시각화에 집중할 수 있다.
10. As a 밴드리 팬, I want to 히스토그램에서 각 티어(최애·차애·호·중간·불호)별 곡 수를 확인할 수 있길 바란다, so that 직관적으로 선호 분포를 파악할 수 있다.

### 공유 기능

11. As a 밴드리 팬, I want to 공유 시 히트맵을 포함할지 선택할 수 있길 바란다, so that 원하는 형태의 결과물을 공유할 수 있다.
12. As a 밴드리 팬, I want to 공유할 티어를 1개 이상 선택할 수 있길 바란다 (예: 최애만, 최애+차애 등), so that 내가 강조하고 싶은 범위만 노출할 수 있다.
13. As a 밴드리 팬, I want to 공유할 밴드를 선택할 수 있길 바란다, so that 특정 밴드의 결과만 공유할 수 있다.
14. As a 밴드리 팬, I want to DC인사이드에 붙여넣기만 하면 자동 렌더링되는 HTML 결과물을 생성할 수 있길 바란다, so that 커뮤니티에서 손쉽게 소팅 결과를 자랑할 수 있다.
15. As a 밴드리 팬, I want to 선택한 티어의 곡들에 대해 유튜브 링크가 자동 포함된 공유 결과물을 받을 수 있길 바란다, so that DC인사이드의 유튜브 링크 자동 렌더링 특성을 활용해 곡을 바로 들을 수 있다.
16. As a 밴드리 팬, I want to 공유 미리보기를 생성 전에 확인할 수 있길 바란다, so that 의도하지 않은 내용이 공유되지 않도록 방지할 수 있다.

## Implementation Decisions

### 소팅 단위 전환
- 데이터 모델을 Album 중심에서 Song 중심으로 재구성한다.
- Song은 밴드 귀속 정보(bandId), 제목, 유튜브 링크, 현재 랭크를 포함한다.
- 밴드 목록은 기존과 동일하게 유지하되, 각 밴드에 속한 Song 목록을 하위 데이터로 관리한다.

### 랭크 카테고리 단순화
- 기존 다단계 티어를 **최애 / 차애 / 호 / 중간 / 불호** 5단계로 통일한다.
- 티어 값은 내부적으로 정수(예: 5~1)로 저장하고, 표시 레이블은 별도 매핑 테이블로 관리한다.

### 인터랙션 플로우
- 밴드 아이콘 클릭 → 해당 밴드 곡 리스트 아코디언 방식으로 펼침
- 곡 클릭 → 랭크 선택 팝업(모달) 등장
- 팝업은 PC와 모바일에서 동일한 컴포넌트를 사용하며, 반응형 레이아웃으로 대응

### 시각화 모듈
- **밴드별 히스토그램**: 각 밴드 섹션 하단 또는 요약 영역에 배치. 5개 티어 × 곡 수를 바 차트로 표현.
- **전체 히트맵**: 별도 탭 또는 결과 요약 화면에서 확인 가능. 밴드(행) × 티어(열) 구조.
- 앨범 자켓의 랭크 표시 영역은 PC/모바일 모두 제거.

### 공유 기능 모듈
- 공유 옵션 UI: 티어 다중 선택 체크박스 + 밴드 다중 선택 + 히트맵 포함 여부 토글
- DC인사이드 HTML 생성기: 선택된 곡 목록과 유튜브 링크를 DC인사이드 호환 HTML 문자열로 직렬화
- 유튜브 링크는 Song 데이터에 저장된 값을 그대로 삽입하며, 링크가 없는 곡은 텍스트만 포함

## Testing Decisions

좋은 테스트는 **내부 구현이 아닌 외부 동작**을 검증한다. 컴포넌트 내부 상태가 아니라 사용자가 보는 결과(화면 변화, 생성된 HTML 등)를 기준으로 작성한다.

### 테스트 대상 모듈

- **랭크 저장 로직**: 곡에 티어를 부여했을 때 올바르게 저장·수정되는지
- **히스토그램 데이터 계산기**: 주어진 Song 배열에서 티어별 카운트를 정확히 집계하는지
- **히트맵 데이터 계산기**: 밴드×티어 매트릭스가 올바르게 생성되는지
- **DC HTML 생성기**: 입력 조건(티어 필터, 밴드 필터, 히트맵 포함 여부)에 따라 예상 HTML 문자열이 생성되는지

### 테스트 전략
- 단위 테스트: 히스토그램/히트맵 계산 함수, HTML 생성 함수
- 통합 테스트(선택): 밴드 클릭 → 곡 리스트 펼침 → 팝업 등장 → 티어 선택 → 저장 플로우

---

## Out of Scope

- 곡 데이터(제목, 유튜브 링크 등) 자동 크롤링 또는 외부 API 연동
- 소팅 결과 서버 저장 및 계정 기반 히스토리 관리
- 다국어(i18n) 지원
- 밴드 또는 곡 추가/삭제 관리자 기능
- 히트맵 디자인 최종 확정 (위치 및 포함 여부 추가 협의 필요)

## Further Notes

- 히스토그램 위치(밴드 섹션 내부 vs. 별도 요약 패널)는 UI 프레임 시안 확정 후 결정 권장
- 히트맵의 최종 유지 여부도 시안 검토 후 결정 (현재 "고민 중" 상태)
- 작업 순서는 기획서 권장 순서를 따름: UI 프레임 → 인터랙션 구조 → 시각화 → 공유 기능
- DC인사이드 유튜브 자동 렌더링 특성에 의존하는 공유 기능이므로, DC 정책 변경 시 재검토 필요
