## 조사 필요 (보류 — 채택/반려 미정)

* **곡 단위 가사 의미공간 2D 클러스터** (구 HANDOFF #3 원안) — 곡=점, 가사 문장 임베딩을 2D 투영. 'Roselia 안의 락발라드' 같은 *밴드 내 이단아 곡*·밴드별 응집/분산도를 보여줌. **키워드 단위 클러스터(현 HANDOFF #3)와는 답하는 질문이 다른 별개 산출물.** 2026-06-26 환원 사유: 이 곡 단위 뷰가 **사용자의 흥미를 실제로 끌 컨텐츠인지 조사 필요**. 키워드 단위 클러스터 출시 후 실제 가사 분포·반응을 보고 채택 판단. (오디오/BPM 불필요, TOP10 가사 재활용 가능.)

## 반려사항

* **장르 자동 추출 / 곡 단위 장르 클러스터링** (아이디어 2 원안) — 오디오 파일이 없고 Spotify Audio Features API가 2024.11부로 신규 앱 차단됨. BPM·음높이 등 음향 특성을 자동 추출할 경로가 사실상 없음. 게다가 2~3개 특성만으론 'J아니메+락'이라는 공통 성향 안에서 세부 록 장르(포크/블루스/프로그레시브/하드/얼터너티브…)를 변별하기 어려움. → 곡 단위 변별 욕구('Roselia 안의 락발라드 잡기')는 **가사 기반 의미공간 2D 클러스터**(HANDOFF 3번, 오디오 불필요)로 대체.

* **곡 단위 메타데이터: 곡 길이·곡 장르·수록 자켓** (아이디어 1 일부) — 곡 길이는 빌드타임 스크랩/IFrame으로 확보 가능하나, 곡 장르는 자동 산출 불가(위와 동일) + 곡별 수동 태깅 부담. 곡 단위 정보 전반이 수집·관리 비용 대비 효용이 낮다고 판단 → 메타패널을 **'밴드 중심'**(밴드소개 + 밴드 워드클라우드)으로 전환하고 곡 단위 메타는 제외. (HANDOFF 2번)

* **곡 단위 워드클라우드** — 곡 1개 가사는 짧아 단어 구름이 빈약하고, 곡별 가사 수집 부담이 큼. 또 곡 단위로 가사를 자동 수집하는 것은 어문저작권상 위험(공식·무료 가사 수집처 부재). → **밴드 단위**(조회수 TOP10 곡 가사 합산, 사용자 직접 제공, 원문 미보관)로 대체. (HANDOFF 2번)
