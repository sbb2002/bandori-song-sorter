# 지시 사항
유튜브의 안티봇 차단 레이더 및 'HTTP Error 429: Too Many Requests'를 원천 차단하기 위해, 아래에 제시된 [기술적 우회 가이드라인]을 모두 충족하는 완벽한 단일 `yt-dlp` Bash 명령어를 작성하라.

# 기술적 우회 가이드라인 (필수 포함 조건)
1. 트래픽 최소화: 데이터 분석 목적에 맞게 영상/자막/썸네일 요청을 차단하고, 순수 오디오 최적 포맷(`-f ba`) 및 mp3 추출 설정 적용
2. 무작위 인간 행동 모방: 곡과 곡 사이의 대기 시간을 30초~60초 사이로 불규칙(Random)하게 부여
3. 내부 API 과부하 방지: 다운로드 전후 메타데이터 파싱 단계에서도 최소 5초간 대기 설정 추가
4. 대역폭 위장: 다운로드 속도를 의도적으로 초당 250KB 수준으로 제한하여 일반 유저의 실시간 음악 스트리밍 패턴으로 위장
5. 세션 신뢰도 확보: 사용자의 Chrome 브라우저 실시간 세션 쿠키를 연동하는 옵션 포함

# 출력 제약 조건
- 오직 가독성이 좋은 멀티라인( `\` 사용 ) 형태의 단일 Bash 명령어만 작성할 것.
- Python, PowerShell 등 Bash 이외의 코드는 절대 포함하지 말 것.
- 각 옵션이 위 가이드라인의 어떤 항목을 해결하는지 명확한 주석이나 테이블 설명을 첨부할 것.

# Bash example
```
yt-dlp \
  -f "ba" -x --audio-format mp3 \
  --sleep-requests 5 \
  --sleep-interval 30 \
  --max-sleep-interval 60 \
  --limit-rate 250k \
  --cookies-from-browser chrome \
  --user-agent "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36" \
  --no-playlist \
  --no-no-overwrites \
  -a list.txt
```
