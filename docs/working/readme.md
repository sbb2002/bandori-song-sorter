# docs/working — 문서 작성 규칙

작업 문서가 구조적으로 유지되도록 한 **HANDOFF.md·done.md·report/의 역할과 작성 규칙**. 새 세션은 이 규칙대로 갱신한다.

## 문서 역할 (한 줄 요약)
| 문서 | 역할 | 시점 | 원칙 |
|------|------|------|------|
| **HANDOFF.md** | **앞으로 할 일의 인덱스** | 미래지향 | 요약 + 링크만. 상세는 여기 두지 않는다 |
| **done.md** | **끝난 일의 아카이브** | 과거 | 세션 단위 append. 완료 상세는 여기 |
| **report/`<주제>`/** | 조사 1건의 작업 산출물 + README | 진행 중~완료 | 데이터·그림·상관표. 재현 명령 포함 |
| **../research/** | 다듬은 연구 논문 | 완료 | 동기→방법→결과→결정. report를 승격 |

핵심: **HANDOFF는 짧게(할 일), done은 자세히(한 일), 깊은 분석은 report/research로 링크.** 같은 내용을 세 곳에 중복하지 않는다.

## 구현 유형별 문서화 경로 (어디에 기록하나)
작업의 성격 = **분석을 몇 번 시도했나**로 기록처가 정해진다.

| 유형 | 기록처 | 무엇을 |
|------|--------|--------|
| **단순 구현** (분석 0회) | **done** | 완료 사실·변경점만 done.md 세션 항목에. |
| **분석이 필요한 구현** (분석 1회) | **done + report** | done에 완료 기록 + `report/<주제>/`에 **분석 결과물(수치·표·플롯·그림)** 과 **간단한 해석**. |
| **2회 이상 분석을 시도한 아이템** | **done + report + research** | 위에 더해, 그 아이템이 **종결되면** done·report를 종합해 **논문 형식**으로 `../research/`에 승격. 그림·표·플롯 **전부 포함**. |

- **report** = "분석 산출물 + 간단한 해석" 수준(진행 중에도 갱신). 무겁게 서술하지 않는다.
- **research** = 종결된 여정(여러 번의 분석 시도·실패·전환)을 정제한 논문. 양식은 [../research/README.md](../research/README.md).
- 애매하면 시도 횟수로 판단: **0회=done · 1회=+report · 2회+ 및 종결=+research.**

## HANDOFF.md 규칙
- **최상단 `마지막 갱신:` 줄** = 이번 세션 한 줄 요약(세션번호·완료작업·main 머지 해시·핵심 결정). 직전 것들은 아래 `이전:` 줄로 강등하고 **5~6개까지만** 유지(더 오래된 건 done.md에 있으니 삭제).
- **`## 우선순위` 표** = 작업별 상태 한눈에(✅/🚧 + 한 줄 + 상세 링크). 완료 시 ✅ + 커밋 해시로 갱신.
- **`## 작업 N` 절** = **요약 + 참고 레퍼런스 링크**만. 구현 상세·수치·판정은 done.md나 report에 두고 링크. (진행 중 작업만 임시로 상세를 둘 수 있으나, 완료되면 done으로 이관.)
- 완료된 작업: 표를 ✅로 바꾸고, 상세는 done.md로 옮기고, 작업 절엔 "done N 참조" 한 줄만 남긴다.

## done.md 규칙
- **세션 1건 = `# 세션 N — 제목 (날짜, 브랜치→main 해시)` 1개.** 시간순 **맨 아래 append**(최신이 끝).
- 세션번호는 직전 done의 최대치 +1. HANDOFF의 `done N` 참조와 번호 일치.
- 구조: **무엇을·왜·파일·결정**. 여러 Phase면 `## Phase` 소절로. 깊은 분석은 report/research 링크(본문에 복붙 금지).

## 공통 규칙
- **절대 날짜**("어제"·"이번 주" 금지 → `2026-07-08`). 상대표현은 다음 세션에 뜻이 바뀐다.
- **커밋 해시·브랜치 명시**(추적 가능). 라이브 반영 여부(main 머지/Pages 배포)도 적는다.
- **한 줄 = 한 사실**. 근거가 길면 report/research로 빼고 링크.
- 데이터/산출물은 report/`<주제>`/에 두고, 전환성 파일(`*_progress.json`·`*_control.json`)은 gitignore(데이터 CSV·결과는 커밋).
- 결론이 "음의 결과"여도 기록한다(왜 안 되는지가 다음 판단의 자산).

## 예시 (세션 30)
- HANDOFF: 표 `5. …` → ✅ + 논문 링크 / 최상단 갱신선 = "작업 5 완료·머지 172684e … 결정: x=timbre·y=valence 유지".
- done.md: `# 세션 30 — 작업 5 … (2026-07-07, …→main 172684e)` + Phase A/B0/C 소절.
- 깊은 방법·수치 = `../research/emotion-axes-extraction.md`, 작업 데이터 = `report/emotion-axes/`.
