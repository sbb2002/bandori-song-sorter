### 3. ux-02 후속: Download 이미지에 밴드별 점수 막대 + 신뢰도 표시 — 미구현 (설계 합의, 시각 프로토타입 대기)
부분평가 시 평가곡이 적은(n≈τ) 밴드는 베테랑이어도 점수가 눌리므로(위 #0 결정의 수용된 귀결), 각 밴드에 점수 막대와 신뢰도를 함께 표시해 "점수 낮음 = 덜 평가함"을 인지하게 함.

**확정 설계 — 2채널 인코딩** (선호도와 표본수를 한 막대에 합성하지 말고 두 시각 채널로 분리; `buildCaptureDOM` 밴드 헤더에 추가):
- **막대 길이 = 선호도 `max(0, R_k)/4`** (R_k=`bandScores().raw`, 0~1). "그 밴드 곡을 얼마나 높게 매겼나" — 단일·직관 개념.
- **막대 투명도 = 신뢰도 `w(n)=1-exp(-n/τ)`** (n=`bandScores().n`). 적게 평가 → 흐림(=불확실). w(n)을 *숫자 아닌 흐림*으로 써서 직관성 확보(불직관 수치 노출 회피).
- **숫자 = 실제 점수 `Score_k`** (=`bandScores().score`=`max(0,R·w)`). 랭킹 진실값.
- (선택) 작은 글씨 `n / 밴드곡수`. 밴드곡수 = `dedupedByBand[band].length`.

**직관 예**: 3곡 전부 최애 → 막대 풀 길이지만 유령처럼 흐림 + 숫자 2.30 = "좋아하지만 아직 불확실"(오해 없음). 다 평가한 최애 → 풀 길이+선명. 다 평가한 평범 → 짧고 선명.

**폐기된 대안(되돌리지 말 것)**:
- ❌ 막대=`w(n)`: 값 불직관 + 밴드별 도달 상한 다름.
- ❌ 막대=완성도 `n/N`: 1곡 밴드가 100%로 떠 신뢰성 착시.
- ❌ 막대=`Score/(4·w(N))`(선호+표본 합성): 한 막대에 두 개념 → "주먹구구"식 불직관 → 2채널로 분리해 해결.

**상태**: 사용자 "참신해서 좋다, 일단 봐야 판단" → **설계 합의, 시각 프로토타입 후 최종 확정.** 프로토타입 시 투명도 매핑(예: `0.15 + 0.85·w(n)`로 최저 가독성 확보 — 기존 히트맵 `0.18+0.82·alpha` 패턴과 동일), 막대 위치·크기·색 함께 결정.

**구현 메모**: `core.bandScores`가 이미 `{score, raw, n}` 반환 → 추가 계산 거의 없음. 투명도용 `confidence(n)=1-exp(-n/τ)` 헬퍼를 core에 추가(테스트 가능, τ=`C.SCORE_TAU`) 고려. 이 표시는 점수·1위 선정을 **바꾸지 않음**(설명 전용).
