# ux-02-ex1: 최애 밴드 스코어링 산출식 설계

## 배경

Download 결과물의 **최애 밴드 선정 및 해당 이미지 출력** 기능에 적용할 스코어링 공식.
현재 `findBestBand()`의 단순 비율 비교(lexicographic sort)를 대체한다.

---

## 확정된 스코어링 공식

### Step 1. 티어별 점수

| 티어 (프로젝트) | 티어 (공식) | 점수 s_t |
|----------------|------------|---------|
| 최애 | Perfect | +4 |
| 차애 | Great   | +3 |
| 호   | Nice    | +2 |
| 보통 | Good    | +1 |
| 불호 | Bad     | -4 |

### Step 2. Raw Score (티어 가중합)

```
R_k = Σ( s_t × c_{k,t} / n_k )
```

- `c_{k,t}`: 밴드 k에서 티어 t로 평가된 곡 수
- `n_k`: 밴드 k의 전체 평가 곡 수
- 범위: -4 ~ +4

예시: 10곡 평가 (최애 5, 호 3, 불호 2)
```
R_k = (+4 × 5/10) + (+2 × 3/10) + (-4 × 2/10) = 2.0 + 0.6 - 0.8 = 1.8
```

### Step 3. 소표본 보정 (Exponential Shrinkage)

```
w(n) = 1 - exp(-n / τ)
```

- n → 0 이면 w → 0 (점수 소멸, 신뢰 불가)
- n → ∞ 이면 w → 1 (보정 없음, 완전 신뢰)
- τ: 보정 강도 파라미터 (아래 τ 결정 참고)

### 최종 공식

```
Score_k = R_k × w(n_k)
           ↑         ↑
      티어 가중합   표본수 보정
```

하한선: `Score_k = max(0, Score_k)`

---

## 설계 의도 및 제약 조건

1. 소표본 밴드(신규 밴드 등)는 점수를 0에 가깝게 수축 → 신뢰도 반영
2. 평가 곡 n ≥ 10 수준이면 보정이 거의 없는 점수로 수렴
3. 불호는 음의 점수; 총점 하한선은 0점
4. 총점 1위 밴드만 이미지 출력

### 엣지 케이스

- **평가 곡 0개**: Score = 0, 이미지 미출력 대상
- **불호 다수 밴드**: Score < 0 → max(0, …) 클램핑 → 평가 0개 밴드와 동일 점수(0)
  - 두 경우를 동일 취급하는 것을 허용하는지 추후 확정 필요

---

## τ 파라미터 결정

"n ≥ 10에서 보정 없음(w ≈ 0.95 이상)" 기준:

```
1 - exp(-10 / τ) ≥ 0.95
τ ≤ 10 / ln(20) ≈ 3.3
```

→ **τ = 3~4 고정값 권장** (전체 평균 표본 수를 쓰면 밴드 간 편차에 따라 보정이 과도하게 약해질 수 있음)

| n  | w(n), τ=3.5 | 비고 |
|----|-------------|------|
| 1  | 0.25 | 신규 밴드, 75% 감점 |
| 3  | 0.57 | 불안정 |
| 5  | 0.76 | 어느 정도 신뢰 |
| 10 | 0.94 | 거의 보정 없음 ✅ |
| 20 | 0.997 | 완전 신뢰 |

## Download 이미지 UI 변경 사항

- 밴드 이름과 같은 줄에 **오른쪽 정렬**로 스코어 표시
- `buildCaptureDOM()` 내 밴드 헤더를 flex row로 교체

```js
// 변경 전
const name = document.createElement('div');
name.textContent = bandDisplay(b);

// 변경 후 (개요)
const nameRow = document.createElement('div');
nameRow.style.cssText = 'display:flex;justify-content:space-between;align-items:baseline;';
// 왼쪽: 밴드명, 오른쪽: score.toFixed(2)
```
