# 플레이리스트 메이커(가칭)의 기획안(초안)

## 개요

- 배경 : 기존 프로젝트 'bandori-song-sorter'의 challengeable한 개발에도 사용자의 시장반응이 시원찮음이 확인됨. 이 앱의 시장에서의 수명한계에 도달한건지, 마케팅 부족으로 인한 어필 감소인지, 사용자 니즈에 부합하지 않아 인기가 식은건지 확인이 필요함. 그러나 그 이유와 별개로 지금 기존 프로젝트를 고도화하는 것은 코스트 대비 효용성이 낮다고 판단됨. 따라서 지금까지 활용 및 추출한 데이터를 바탕으로 새로운 어트랙션 앱을 개발하고자 함.

- 목표 : 사용자 재방문성이 높은 앱을 만들기

- 핵심아이템 : 플레이리스트 메이커, 플레이리스트 공유, 사용자 친화적 앱

## 상세 내용

- 필요 데이터 : 유튜브 url DB(유튜브 재생을 위해 필요), 오디오 캐시(분위기 매칭 분석을 위해 필요)

- 필요 API : 유튜브 api, open-router api(사용자의 자연어 요구에 따라 필요), umami(사용자 통계 분석을 위해 필요)

- 핵심 기능 : 사용자가 '오늘 퇴근하면 주말 시작이야~ 기분이 절로 좋아지는 플레이리스트를 만들어줘.'와 같은 자연어 요청을 보냄. 그러면 open-router api를 통해 자연어를 분석하고 밝은 느낌이 나는 곡을 1차 선택 후 플레이리스트 에너지 진행 단계(사용자 설정 가능)에 따라 1단계에서 지정된 에너지 레벨과 같은 노래를 선택한 후 (다음 에너지 단계) & (화성학적으로 이어지는 느낌이 드는) 노래를 다음 곡으로 선택하여 약 1시간(사용자 요청에 따라 변경 가능) 정도의 플레이리스트를 생성. 플레이리스트 생성 후 유튜브 프레임을 통해 순서대로 재생할 수 있고, 이 플레이리스트를 '유튜브 재생목록'으로 만들어 사용자의 계정에 저장하거나 다른 사람에게 공유할 수 있음.

- 부수 기능 : 

    1) 설정 버튼을 통해 아래 기능들을 제공하고자 함. UI는 아직 미정.
        * 에너지 레벨 조절 기능 : 플레이리스트를 시간 순으로 N개의 단계로 나눌 수 있고 각 단계마다 지정된 시간 또는 일정 곡 수, 해당 단계의 에너지 레벨을 지정할 수 있음.
        * 밴드 체크박스 기능 : 특정 밴드를 1개 이상 선택하여 해당 밴드 내에서만 검색할 수 있음. ALL을 선택하거나 체크박스가 모두 비면 ALL로 선택되어 모든 밴드를 대상으로 검색함. 이는 각 에너지 구간별로도 설정 가능.

    2) 서빙 시 사용자가 몰리게 되면 open-router의 ratelimit에 걸릴 위험이 있음. 따라서 큐 제어가 필요함.

    3) 큐 제어와 open-router api key 값 관리를 위해서 아마 백엔드 서버가 필요할거라 생각됨. 될 수 있는 한 무료 서버가 좋음. 아무도 이용안할 때는 sleep해도 괜찮음.

[참고] 핵심 기능은 파일럿 테스트 시에 필수적으로 구현되어야 하는 기능임. 부수 기능은 파일럿 테스트 통과 시 부가적으로 업데이트할 사항임.
